Guia de Configuração: Túnel Cloudflare Fixo (Zero Trust)
Pré-requisitos
Um domínio próprio registrado (ex: mygames.com.br).

Uma conta gratuita na Cloudflare com o domínio já configurado e apontado para ela.

Ambiente Docker (docker-compose.yml) rodando os aplicativos localmente nas portas internas 5000 (Site) e 5002 (Admin).

Passo 1: Criar o Túnel e Gerar o Token
Esta etapa substitui os links dinâmicos (trycloudflare.com) por uma conexão permanente e autenticada com a sua conta Cloudflare.

1. Acesse o Painel da Cloudflare e faça login.
2. No menu lateral esquerdo, clique em Zero Trust.
3. Dentro do painel Zero Trust, navegue até Networks > Tunnels.
4. Clique no botão Create a tunnel.
5. Selecione o conector do tipo Cloudflared e clique em Next.
6. Dê um nome ao túnel que identifique o ambiente (ex: mygames-homologacao ou mygames-producao) e salve.
7. Na tela "Install and run a connector", escolha a aba Docker.
8. A Cloudflare exibirá um comando de instalação longo. Copie apenas a sequência de caracteres gigante que aparece logo após o 
   parâmetro --token. (Guarde este token de forma segura, ele será usado no docker-compose.yml).

Passo 2: Criar as Rotas Públicas (Public Hostnames)
Ainda no painel da Cloudflare (após copiar o token, clique em Next), você definirá quais URLs da internet vão apontar para quais portas 
dentro do seu Docker.

Rota do Site do Cliente:
- Subdomain: site-homolog (ou deixe em branco para o domínio principal em produção)
- Domain: seudominio.com.br
- Service Type: HTTP
- URL: 127.0.0.1:5000
- Clique em "Save hostname".

Rota do Painel Admin (Adicione um novo Public Hostname):
- Subdomain: admin-homolog
- Domain: seudominio.com.br
- Service Type: HTTP
- URL: 127.0.0.1:5002
- Clique em "Save hostname".

Passo 3: Atualizar o docker-compose.yml
No seu servidor ou desktop, você precisa alterar o comportamento dos contêineres da Cloudflare para que eles parem de pedir links aleatórios 
e passem a usar o seu Token Fixo.

Substitua a configuração dos túneis no seu arquivo docker-compose.yml pelo bloco abaixo:

# ----------------------------------------
  # TÚNEL CLOUDFLARE - SITE (COM TOKEN FIXO)
  # ----------------------------------------
  cf-tunnel-site:
    image: cloudflare/cloudflared:latest
    container_name: tunnel_site
    network_mode: "service:mygames-site"
    command: tunnel run
    environment:
      - TUNNEL_TOKEN=COLE_SEU_TOKEN_GIGANTE_AQUI
    restart: unless-stopped
    depends_on:
      - mygames-site

  # ----------------------------------------
  # TÚNEL CLOUDFLARE - ADMIN (COM TOKEN FIXO)
  # ----------------------------------------
  cf-tunnel-admin:
    image: cloudflare/cloudflared:latest
    container_name: tunnel_admin
    network_mode: "service:mygames-admin"
    command: tunnel run
    environment:
      - TUNNEL_TOKEN=COLE_SEU_TOKEN_GIGANTE_AQUI
    restart: unless-stopped
    depends_on:
      - mygames-admin

Aviso de Segurança: É recomendado não deixar o token diretamente escrito no arquivo .yml em ambientes de produção. O ideal é usar 
${TUNNEL_TOKEN} e colocar o valor real dentro do seu arquivo .env.

Passo 4: Fazer o Deploy e Validar
Com o arquivo atualizado, reinicie o ambiente Docker para aplicar as mudanças.

1. Execute o script de deploy ou os comandos manuais:
docker compose down
docker compose up -d

2. Volte ao painel Zero Trust > Tunnels na Cloudflare.

3. O status do seu túnel deverá mudar de "Inactive" para "Healthy" (Verde).

4. Acesse as URLs oficiais (ex: https://site-homolog.seudominio.com.br) no navegador para testar. O certificado SSL/HTTPS já estará 
configurado automaticamente.
